Creating Clusters and Nodes using yaml

Here's an example of a YAML file for creating a Kubernetes cluster:

apiVersion: kind.x-k8s.io/v1alpha4
kind: Cluster
name: my-cluster
nodes:
- role: control-plane
- role: worker
- role: worker
networking:
  apiServerAddress: "0.0.0.0"
  apiServerPort: 6443
  podSubnet: "10.244.0.0/16"
  serviceSubnet: "10.96.0.0/12"

This YAML file defines a Kubernetes cluster with one control-plane node and two worker nodes. It also specifies networking configurations including API server address, port, and subnet ranges for pods and services.

Namespace

Create a Namespace

kubectl create namespace <my-namespace>

Get All Namespaces

kubectl get namespaces

Get Specific Namespace 

kubectl get namespace <my-namespace>

Delete a Namespace

kubectl delete namespace <my-namespace>

Note: Deleting a namespace will also delete all resources within that namespace, so use this command with caution.

Abbreviation for Namespace in Kubernetes

The common abbreviation for namespace in Kubernetes is "ns".

This abbreviation is often used in kubectl commands and YAML manifests. For example:

kubectl get ns

This command will list all namespaces in the cluster, using the abbreviated form "ns" instead of "namespace".

API-RESOURCES

API resources in Kubernetes are the different types of objects that can be created, modified, and managed within a Kubernetes cluster. These include pods, services, deployments, and many others.

View All API Resources

kubectl api-resources

View API Resources with More Details

kubectl api-resources -o wide

View API Resources for a Specific API Group

kubectl api-resources --api-group=apps

View Namespaced API Resources

kubectl api-resources --namespaced=true

View Non-Namespaced API Resources

kubectl api-resources --namespaced=false

These commands help you explore and understand the various resources available in your Kubernetes cluster, which is crucial for effective cluster management and troubleshooting.

You can sort the output of the api-resources command using the '--sort-by' flag. Here are some examples:

Sort API Resources by Name

kubectl api-resources --sort-by=name

Sort API Resources by Kind

kubectl api-resources --sort-by=kind

Sort API Resources by API Group

kubectl api-resources --sort-by=group

These sorting options can help you organize the output of api-resources for easier viewing and analysis.

Get All Details in a Namespace

To get all details in a specific namespace in Kubernetes, you can use the following command:

kubectl get all -n <namespace-name>

This command will display all resources (pods, services, deployments, etc.) in the specified namespace. If you want even more detailed information, you can add the '-o wide' flag:

kubectl get all -n <namespace-name> -o wide

For a complete, detailed output in YAML format, you can use:

kubectl get all -n <namespace-name> -o yaml

Remember to replace <namespace-name> with the actual name of the namespace you want to inspect.
